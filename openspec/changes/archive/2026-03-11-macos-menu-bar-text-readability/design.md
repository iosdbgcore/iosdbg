## Context

当前调试器使用 egui 框架和 x64dbg 风格的深色主题。顶部控制面板和底部状态栏中的文字颜色对比度不足，特别是在 macOS 深色模式下。现有颜色值（RGB 120-210）在深色背景（RGB 26,30,36）上可读性较差。

## Goals / Non-Goals

**Goals:**
- 提升控制面板和状态栏文字的对比度至 WCAG AA 标准（4.5:1）
- 保持 x64dbg 主题的视觉风格和色相
- 最小化代码变更，仅调整颜色值

**Non-Goals:**
- 不修改寄存器面板和内存查看器（已有足够对比度）
- 不改变主题系统架构
- 不添加可配置的主题切换功能

## Decisions

### Decision 1: 直接修改硬编码颜色值而非更新调色板

**选择**: 在 `control_panel.rs` 和 `status_bar.rs` 中直接调整 `Color32::from_rgb()` 的参数值

**理由**:
- 当前调色板（`UiPalette`）仅定义了 8 个基础颜色，不覆盖所有 UI 组件的具体用色
- 各组件使用的颜色值是硬编码的，不引用调色板
- 直接修改可以精确控制每个文字元素的对比度

**替代方案**: 扩展 `UiPalette` 添加 `text_control_panel`、`text_status_bar` 等字段
- 被拒绝原因：需要重构所有组件以引用新字段，变更范围过大

### Decision 2: 使用亮度提升策略而非完全重新选色

**选择**: 保持现有颜色的色相和饱和度，仅提升 RGB 各分量至 >= 200

**理由**:
- 保持语义颜色的可区分性（绿色=成功，红色=错误，橙色=警告）
- 最小化视觉冲击，用户无需重新适应配色
- 数学上简单：`new_rgb = max(old_rgb, 200)` 或按比例缩放

**替代方案**: 使用 HSL 色彩空间调整亮度（L 值）
- 被拒绝原因：egui 使用 RGB，需要额外的色彩空间转换逻辑
