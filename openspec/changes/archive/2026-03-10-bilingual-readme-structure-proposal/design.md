## Context

项目是一个基于 Rust + LLDB + egui 的可视化调试器，已有完整的 OpenSpec 技术规范（9 个 spec 文档），但缺少用户友好的入口文档。目标用户包括中英文开发者。

## Goals / Non-Goals

**Goals:**
- 创建结构清晰的中英双语 README
- 覆盖项目介绍、安装、使用、贡献等标准章节
- 链接到现有技术文档，避免重复

**Non-Goals:**
- 不重写或替代现有 OpenSpec 技术文档
- 不添加自动化翻译工具
- 不修改项目代码或构建流程

## Decisions

### 决策 1：双文件结构 vs 单文件双语
**选择**：使用 `README.md`（中文）+ `README.en.md`（英文）双文件结构

**理由**：
- GitHub 默认显示 README.md，中文用户优先
- 双文件避免单文件过长，提升可读性
- 便于独立维护和更新

**替代方案**：单文件内嵌中英文 → 被拒绝（文件过长，阅读体验差）

### 决策 2：README 章节结构
**选择**：采用标准开源项目结构

```
1. 项目简介（带 logo/截图）
2. 功能特性
3. 安装指南（依赖 + 构建步骤）
4. 快速开始（最小示例）
5. 使用文档（链接到 OpenSpec）
6. 贡献指南
7. 许可证
```

**理由**：符合开源社区习惯，降低学习成本

### 决策 3：内容深度
**选择**：README 提供概览，详细文档链接到 `openspec/specs/`

**理由**：避免文档重复维护，保持单一信息源

## Risks / Trade-offs

**[风险] 双语文档同步维护成本** → 缓解：在贡献指南中明确要求同步更新两个文件

**[权衡] 中文优先 vs 英文优先** → 选择中文优先（README.md），因为项目初期主要面向中文用户
